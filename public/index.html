State in React

React is a very flexiable library, but in the last lesson on "React Components and Props", we ended on that React has a single strict rule:

All React components must act like pure functions with respect to their props.

function withdraw(account, amount) { account.total -= amount; }

There was this function we looked at that is considered unpure.

If this were a valid react component it would look like this:

function withdraw(props) { props.account.total -= props.amount; }
